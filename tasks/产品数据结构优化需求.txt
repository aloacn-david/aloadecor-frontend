================================================================================
产品数据结构 & Shopify API 优化需求说明
================================================================================
创建日期：2026-03-20
更新日期：2026-03-20

================================================================================
一、核心目标
================================================================================

1. Shopify 作为唯一的产品数据来源（Source of Truth）
2. Shopify API 只调用一次（用于同步），不允许在不同模块重复调用
3. 产品链接、产品内容、产品状态等必须基于同一份产品数据
4. 前端不允许直接调用 Shopify API，只能调用我们自己的后端接口

================================================================================
二、当前问题
================================================================================

目前系统中存在以下潜在问题：
- 不同模块（产品信息 / 产品链接 / 产品内容）可能分别调用 Shopify API
- 数据来源不统一，容易出现重复请求
- 页面性能下降
- 后续扩展（内容管理、状态管理）会变得混乱

================================================================================
三、整体解决方案
================================================================================

1. 建立统一的 Shopify 数据同步机制（只调用一次）
----------------------------------------
在后端建立一个 Shopify Sync 模块：
- 负责从 Shopify API 拉取产品数据
- 将数据写入本地数据库
- 支持两种方式：
  - 手动触发同步（按钮）
  - 定时同步（如每天一次）

👉 注意：
Shopify API 只允许在这个模块中调用，其他地方禁止调用。

2. 建立本地产品数据结构（核心）
----------------------------------------
数据必须拆成两层：

（1）产品基础数据表（products）
来源：Shopify
字段建议包括：
- shopify_product_id
- title
- handle
- sku
- price
- images
- variants
- inventory
- status
- updated_at

（2）产品扩展数据表（product_meta）
来源：我们后台维护
字段包括：
- product_id（关联 products）
- 各平台链接（Amazon / Wayfair / Home Depot 等）
- 产品内容（描述 / 文案）
- 内部备注
- 状态（未完成 / 审核中 / 已上线）
- 产品分类（标品 / 半标品 / 非标品）
- 标签（爆品 / 主推等）

3. 提供统一产品接口（给前端）
----------------------------------------
前端只允许调用我们自己的 API，例如：
- GET /api/products
- GET /api/products/:id

接口返回内容必须是：
👉 products + product_meta 合并后的数据

================================================================================
四、数据流逻辑（必须按这个执行）
================================================================================

初始化或同步时：
Shopify API → 后端同步模块 → 本地数据库（products）

后台编辑时：
后台 → 写入 product_meta（不改 Shopify 数据）

前端展示时：
前端 → 调用 /api/products → 后端从数据库读取 → 返回合并数据

================================================================================
五、关键限制（必须遵守）
================================================================================

❌ 不允许前端直接调用 Shopify API
❌ 不允许多个模块分别调用 Shopify API
❌ 不允许产品链接 / 产品内容各自独立数据源

✅ Shopify 只作为"同步源"
✅ 所有业务数据必须基于本地数据库
✅ 产品所有信息必须统一从一个接口输出

================================================================================
六、预期效果
================================================================================

完成后应达到：
- Shopify API 调用次数大幅减少
- 产品数据结构统一，避免重复调用
- 产品链接、内容、状态全部基于同一数据源
- 后续可以方便扩展：
  - AI分析模块
  - 产品状态管理
  - 多平台数据联动

九、新增API端点
================================================================================

统一产品API（从数据库读取）：
----------------------------------------
- GET /api/products - 获取所有产品（基础数据+扩展数据合并）
- GET /api/products/:product_id - 获取单个产品
- POST /api/products/sync - 手动触发Shopify产品同步
- POST /api/products/:product_id/meta - 更新产品扩展数据

================================================================================
十、数据流
================================================================================

同步流程：
1. 调用 POST /api/products/sync
2. 后端从Shopify API拉取产品（只调用一次）
3. 数据写入 products 表
4. 返回同步统计

前端读取流程：
1. 调用 GET /api/products
2. 后端从数据库读取 products 和 product_meta
3. 合并数据后返回
4. 不直接调用Shopify API

后台编辑流程：
1. 调用 POST /api/products/:product_id/meta
2. 数据写入 product_meta 表
3. 不修改Shopify数据
