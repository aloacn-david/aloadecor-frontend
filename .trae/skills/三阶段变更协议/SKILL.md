# AI安全开发协议（AloaDecor Dev Protocol）

适用于：
代码、脚本、网站、自动化工具、Git、部署、服务器等任务。

默认模式：只读分析模式

------------------------------------------------

【最高安全规则】

在用户没有明确授权前，禁止：

- 修改代码
- 创建或删除文件
- 运行终端命令
- 执行自动化脚本
- git add
- git commit
- git push
- 部署
- 修改数据库
- 调用可能改变系统状态的工具

只有当用户明确输入：

Proceed

才允许进入执行流程。

------------------------------------------------

# 第一阶段：问题分析（Analysis）

只允许输出：

1 问题定位  
2 根本原因  
3 涉及文件  
4 风险评估  
5 是否可能影响其他模块  

禁止：

修改文件  
运行命令  
生成补丁  

阶段结束必须输出：

Analysis complete.

Reply **Proceed** to enter solution planning.

------------------------------------------------

# 第二阶段：方案设计（Planning）

输出：

1 解决方案步骤  
2 每一步将修改的文件  
3 修改目的  
4 风险点  
5 回滚方案  

禁止：

修改代码  
运行命令  
git操作  

阶段结束必须输出：

Plan complete.

Reply **Proceed** to execute.

------------------------------------------------

# 第三阶段：执行（Execution）

只有收到 **Proceed** 才允许执行。

执行规则：

- 每次只修改一个问题
- 修改后必须展示 diff
- 修改后进行自检
- 如果出现异常立即停止

------------------------------------------------

# 第四阶段：验证（Verification）

执行后必须进行：

1 编译检查
2 逻辑检查
3 依赖检查
4 是否影响其他模块

------------------------------------------------

# Git 安全策略

默认禁止：

git add  
git commit  
git push  

即使进入执行阶段，也只能：

修改文件  
展示 diff  

只有用户明确说：

允许提交
或
允许 push

才允许执行 Git 操作。

------------------------------------------------

# 自动修复禁止规则

AI 不允许：

自动修复所有问题  
自动批量修改  
自动重构整个文件  

所有改动必须：

小步  
可回滚  
可解释

------------------------------------------------

# 冲突优先级

即使用户说：

"帮我修复"
"直接改一下"

如果用户没有明确说：

Proceed

AI 仍然必须停在分析阶段。

------------------------------------------------

# 输出格式规则

第一阶段结束必须输出：

Analysis complete.
Reply **Proceed** to continue.

第二阶段结束必须输出：

Plan complete.
Reply **Proceed** to execute.

如果没有收到 Proceed，不允许继续。
